================================================================================
                    DATA TRANSFER ISSUE - ROOT CAUSE & FIX
================================================================================

SYMPTOM:
--------
When clicking the heatmap button, not all data from "Data Transfer Sheet" 
was being transferred to "HeatMap Sheet".

ROOT CAUSE:
-----------
Two functions were INCORRECTLY filtering out columns with "DR" in the header:
  - CollectHeaders (line 269-271)
  - CollectHeaderCols (line 285-287)

The filter condition was:
  If Trim$(ws.Cells(anc.row, c).Value) <> "" _
     And UCase$(ws.Cells(anc.row, c).Value) <> "DR" Then  ← WRONG!

WHY THIS WAS WRONG:
-------------------
1. "DR" columns (DR1, DR2, DR3, etc.) contain VEHICLE DATA
2. These functions were SKIPPING all DR columns
3. This resulted in an empty or incomplete collection of source data
4. The data transfer loop had no columns to copy from
5. Result: NO DATA or INCOMPLETE DATA transferred

THE FIX:
--------
Removed the incorrect filter:
  If Trim$(ws.Cells(anc.row, c).Value) <> "" Then  ← CORRECT!
     ' Now includes DR columns

ADDITIONAL IMPROVEMENT:
-----------------------
Added a warning when source has more data than destination can hold:
  If sVehHdr.count > tVehCols.count Then
      MsgBox "Warning: Data Transfer Sheet has X vehicles, but 
              HeatMap Sheet can only accommodate Y vehicles.
              Only the first Y vehicles will be transferred."
  End If
